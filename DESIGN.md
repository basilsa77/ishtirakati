# تصميم اشتراكاتي v12

## الاستكشاف

### 1. جدول التجديدات

كل اشتراك يظهر في قائمة زمنية مباشرة؛ التاريخ والمبلغ والخدمة مرئية في صف واحد، مع تقويم شهري اختياري للمراجعة البصرية.

```text
┌──────────────────────────┐
│  مساء الخير       بحث ◉ │
│                          │
│       ·      ●           │
│    ◌    ١٢ شهرًا   ◉     │
│       8,420 ر.س          │
│  ●          ·            │
│                          │
│ هذا الأسبوع   التسرّب    │
│ Netflix  55   1,140 ر.س │
└──── الرئيسية · مكتبتك ──┘
```

### 2. خريطة الكتل

مساحة كل خدمة تساوي تكلفتها السنوية، لتظهر الخدمات الكبيرة فورًا بلا قراءة أرقام كثيرة.

```text
┌──────────────────────────┐
│ خريطة التزامك السنوي     │
│ ┌───────────┬──────┐     │
│ │ STC       │Claude│     │
│ │           ├───┬──┤     │
│ │           │Nfx│  │     │
│ └───────────┴───┴──┘     │
│ اسحب كتلة للمراجعة       │
└──────────────────────────┘
```

### 3. واجهة الأوامر

لا وجهات ثابتة؛ يبدأ التطبيق ببحث وأوامر مثل «أرني تجديدات الأسبوع» و«أضف اشتراكًا».

```text
┌──────────────────────────┐
│ ماذا تريد أن تفعل؟       │
│ ┌──────────────────────┐ │
│ │ ابحث أو نفّذ أمرًا…  │ │
│ └──────────────────────┘ │
│ القادم  •  غير المستخدم │
│ + إضافة   ↑ استيراد      │
└──────────────────────────┘
```

## الاتجاه المختار

اختير **جدول التجديدات** لأنه يطابق النموذج الذهني المعتاد للمواعيد ويجعل المقارنة بين التاريخ والمبلغ فورية. استُخدمت واجهة الأوامر كطبقة مساعدة، بينما تبقى الإعدادات وجهة مستقلة وواضحة في التنقل.

## نقد ذاتي

كانت النسخة الأولى من الخطة تحتوي صفًا من بطاقات KPI يشبه أي لوحة مالية. استُبدلت بملخص تجديدات مباشر، وشريط «تسرّب صامت» خطي، وقائمة تجديدات بلا حاويات متداخلة. أُبقيت الأرقام الثانوية هادئة حتى لا تنافس القرار الأساسي.

## نظام الرموز

### الألوان

| الرمز | فاتح | داكن | الاستخدام |
|---|---|---|---|
| `ink` | `#13231D` | `#F2F7F4` | النص الأساسي |
| `canvas` | `#F4F7F3` | `#0D1411` | خلفية التطبيق |
| `surface` | `#FFFFFF` | `#151F1A` | سطح الأدوات |
| `pulse` | `#007D5C` | `#47D7A5` | الهوية والحالة النشطة |
| `amber` | `#D79A2B` | `#F0C56A` | موعد قريب/تنبيه |
| `coral` | `#C94F5C` | `#FF8D97` | تسرّب أو خطر مالي |

التباين بين `ink` و`canvas` يتجاوز 4.5:1، ولا يعتمد أي معنى على اللون وحده.

### الخطوط

- العرض: **Tajawal** بوزن 400/700، لأنه هندسي وواضح في الأرقام العربية.
- النص: **IBM Plex Sans Arabic** بوزن 400/600، لارتفاع سطر مريح وكثافة جيدة.
- الخطان مضمنان محليًا مع رخصة OFL ولا يجلب التطبيق خطوطًا من الشبكة؛ تبقى (`SF Arabic`, `Geeza Pro`, `Noto Sans Arabic`) fallbacks للنظام.

### المقاييس

- المسافات: `4, 8, 12, 16, 24, 32, 48`.
- الخط: `12, 14, 16, 20, 28, 36` مع احترام Text Scale.
- أنصاف الأقطار: `4, 8, 14`؛ ويُحجز `14` لملخص التجديدات ولوحة الأوامر.
- أهداف اللمس: `44pt` كحد أدنى.
- الطبقات: حد 1px وظل واحد منخفض؛ لا توجد بطاقات داخل بطاقات.

## العنصر التوقيعي

**جدول التجديدات**: القائمة الزمنية هي العرض الافتراضي؛ تُجمع التجديدات حسب التاريخ وتعرض الخدمة والمبلغ بصفوف ذات أهداف لمس واسعة. التقويم الشهري عرض اختياري، ولا يستخدم لتصفح البيانات الأساسية.

## الحركة

تستخدم الواجهة انتقالات ضمنية قصيرة 180ms فقط، وتصبح فورية عند `disableAnimations`.

## التخطيط والتنقل

- الهاتف: محتوى ممتد ثم `CycleDock` رمزي غير عائم، مع زر أوامر مركزي.
- iPad: شريط أوامر جانبي بعرض ثابت ومحتوى ثنائي الأعمدة.
- لوحة الأوامر تصل إلى جميع الوجهات والإضافة والاستيراد والإعدادات.
- لا يستخدم v12 `BottomNavigationBar` أو `NavigationBar`.

## الميزات المختارة

- لوحة التسرّب المالي والتكلفة السنوية الحقيقية.
- جدول التجديدات والتنبيهات المحلية الحالية بمنطق المنطقة الزمنية الموجود.
- إبراز الاشتراكات بلا استخدام وتقسيم التكلفة العائلية.
- لوحة أوامر وبحث موحدة.
- تخطيط iPad متكيف وتحليلات CustomPainter الحالية.

## ما استُبعد

- النسخ الاحتياطي اليدوي: مستبعد احترامًا لقرار المنتج السابق وتقليلًا لمسارات تسريب البيانات؛ تبقى المزامنة الحالية بإفصاحها الأمني.
- Widget/شاشة القفل: يحتاج App Group وامتدادًا أصليًا ودورة مراجعة مستقلة، ولا يُمزج مع إعادة تصميم حرجة.
- مقارنة سنة بسنة: لا يوجد دفتر حركات تاريخي كامل؛ عرض رقم مصطنع سيكون مضللًا.
