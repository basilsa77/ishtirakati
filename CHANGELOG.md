## 15.2.5+40 - 2026-07-14

### إنشاء أول نسخة سحابية على الجهاز الحقيقي

- عند المراجعة المحلية صفر، يتحقق التطبيق من غياب `users/{uid}` ثم ينشئه مباشرة عبر `set` غير مدمج بالمراجعة 1؛ لا يطلب من المستخدم الضغط مرة ثانية.
- إذا أعاد `transaction.get` خطأ `not-found` خاصًا بالمستند، ينتقل المسار نفسه إلى الإنشاء المباشر، بينما يظل غياب قاعدة `(default)` خطأ إعداد ولا يُخفى.
- التحديثات اللاحقة تظل داخل معاملة Firestore مع مقارنة المراجعات، وقواعد الإنتاج تمنع الإنشاء لمستخدم آخر أو الكتابة فوق مراجعة موجودة.
- البناء الداخلي يعرض العملية ورمز Firebase ووجود المستند والمراجعة وبصمة commit دون UID أو البريد أو الحمولة المشفرة أو مفتاح AES.
- صفحة «حول التطبيق» تعرض الإصدار ورقم البناء وبصمة commit منفصلة، واسم IPA يتضمن بصمة الكود لمنع تثبيت ملف قديم بالخطأ.
- إضافة workflow يدوي مستقل لنشر قواعد وفهارس Firestore إلى `ishtirakati-260f7` باستخدام حساب خدمة محفوظ في GitHub Secrets فقط.

## 15.2.2+37 - 2026-07-14

- ترحيل آمن للنسخ السحابية القديمة: تُقرأ بصيغتها السابقة ثم تُستبدل بنسخة مشفرة بعد نجاح الاستيراد، دون تغيير سجل الجهاز أو مفاتيح Keychain.
- تحديد قاعدة Firestore الافتراضية `(default)` صراحة والتحقق في CI من project ID وapp ID وsender ID وstorage bucket الإنتاجية.

- رسائل عربية منفصلة لانتهاء الجلسة، وانقطاع الشبكة، وتعطل الخدمة، ورفض الصلاحية، وغياب قاعدة Firestore، وقيود التطبيق، وأخطاء Storage إن أضيفت الخدمة مستقبلًا.
- قواعد Firestore تمنع إنشاء أو تحديث نسخة مكشوفة، وتحصر القراءة والكتابة والحذف في صاحب `users/{uid}`، وتمنع الرجوع من الصيغة المشفرة إلى القديمة.
- اختبارات تشفير واستعادة وفشل Keychain، واختبارات محاكي لإنشاء أول مستند والتعارض والعزل والترحيل والانقطاع.
