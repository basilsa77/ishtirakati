# لوحة تحكم المشرف — اشتراكاتي

لوحة كاملة محمية لإدارة التطبيق: المستخدمون والنشاط الفعلي، إشعارات للمستخدمين، كتالوج الخدمات، وقائمة الإطلاق — دون المساس بالتشفير E2E أو قواعد Firestore.

## التشغيل

```bash
npm install firebase-admin
node admin/server.mjs path/to/serviceAccount.json
# ثم افتح http://127.0.0.1:8791
```

مفتاح حساب الخدمة من: Firebase Console ← إعدادات المشروع ← حسابات الخدمة ← إنشاء مفتاح خاص. **لا ترفعه إلى Git** (مضاف إلى .gitignore).

عند أول تشغيل يطلب الخادم إنشاء كلمة مرور للوحة (12 حرفًا فأكثر).

## الحماية

- كلمة مرور مخزنة بـ scrypt + ملح عشوائي في `admin/.admin-auth.json` (خارج Git، صلاحيات 600).
- جلسات HttpOnly + SameSite=Strict تنتهي بعد 30 دقيقة خمول، وخروج يدوي.
- قفل 15 دقيقة بعد 5 محاولات دخول فاشلة.
- الخادم يستمع على `127.0.0.1` حصرًا — لا يمكن الوصول إليه من الشبكة أو الإنترنت، فلا يصل إليه أحد غيرك على جهازك.
- ترويسات CSP وX-Frame-Options وnosniff وno-store + فحص Origin ضد CSRF.
- بيانات اشتراكات المستخدمين تبقى مشفّرة E2E (AES-256-GCM بمفتاح Keychain) — الخادم لا يفكها ولا يستطيع.

## الجديد في اللوحة v2

- **سجل تدقيق** ملحق فقط في `admin/audit.log` (خارج Git): كل عملية إدارية تُسجَّل بالوقت والعملية والمورد والنتيجة وIP وRequest ID — مع تبويب بحث وتصدير، ولا يمكن تعديله من الواجهة.
- **إجراءات المستخدمين:** تعطيل/تفعيل، إبطال الجلسات، وحذف نهائي (Auth + Firestore) يتطلب كتابة بريد الحساب + إعادة إدخال كلمة مرور اللوحة.
- **نشر الإعلانات بمراحل:** مسودة (لا تصل للمستخدمين) ← نشر بكلمة المرور مع نسخة احتياطية تلقائية في `admin/backups/` ← استرجاع Rollback بزر.
- **صحة النظام:** فحص حي لـ Auth وFirestore بزمن الاستجابة + حالة الملفات والخادم.
- **تشديد:** Rate limiting عام (240 طلبًا/دقيقة)، منع تكرار إرسال push خلال دقيقة، Request ID في كل خطأ.
- **وضع داكن** كامل مع حفظ التفضيل.

## الأقسام

- **المستخدمون والنشاط:** كل الحسابات من Firebase Auth مع آخر دخول (lastSignIn)، آخر استخدام (lastRefresh)، آخر مزامنة وعدد المزامنات وحجم النسخة من Firestore. بحث وفرز وتصدير CSV.
- **الإشعارات:** انشر إعلانًا (عنوان، نص، رابط) يُكتب في `catalog/announcements.json`؛ ادفعه إلى GitHub فيجلبه التطبيق بنفس آلية الكتالوج ويعرضه للمستخدمين. خدمة التطبيق جاهزة في `lib/services/remote_announcements.dart` (مع اختبارات) — اربطها بواجهة النبض ليظهر الإعلان. للإشعارات الفورية push يلزم FCM + APNs في إصدار قادم.
- **كتالوج الخدمات:** محرر كامل مع تحقق وتصدير يرفع النسخة تلقائيًا.
- **الإصدار والإطلاق:** مقارنة النسخة بالمنشورة على GitHub + قائمة فحص App Store.

يمكن أيضًا فتح `admin/index.html` مباشرة دون خادم — تعمل أقسام الكتالوج والإطلاق فقط.
